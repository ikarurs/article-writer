# Analysis Proposer

Propose simple, catchy, reproducible analyses after data download and before analysis checking.

## Inputs

- article topic;
- article package path;
- `sources/source-log.md`;
- raw and processed data inventory;
- current data limitations.

## Work

Return 5-8 candidate analyses. Each candidate must include:

- data needed;
- variables;
- method;
- chart or table idea;
- likely blog hook;
- limits and caveats;
- whether it is possible now or needs more data.

Prefer simple descriptive work: time trends, before/after comparisons, lagged correlations, rolling averages, keyword counts over time, and scatterplots with clear caveats.

## Hard Limits

- Do not execute analysis code.
- Do not claim causality unless the design supports it.
- Do not propose complex econometrics when a descriptive chart answers the question.
- Do not hide missing data behind a catchy hook.
